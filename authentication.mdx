## API Token Usage

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
curl -X GET \
  'https://api.karmixalchemist.com/v1/projects' \
  -H 'Authorization: Bearer YOUR_API_TOKEN' \
  -H 'Content-Type: application/json'
  ```
  ```javascript
const response = await fetch('https://api.karmixalchemist.com/v1/projects', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${process.env.KARMIX_TOKEN}`,
    'Content-Type': 'application/json'
  }
});
const projects = await response.json();
  ```
</Request>

## User Roles and Permissions

Karmix Alchemist implements role-based access control to manage permissions effectively.

| Role | Permissions | Description |
|------|-------------|-------------|
| Admin | Full access | Manage projects, users, and system settings |
| Editor | Edit content | Create and modify documentation |
| Viewer | Read access | View documentation without editing |
| Contributor | Limited edit | Edit assigned sections only |

<ExpandableGroup>
  <Expandable title="Admin Permissions" default-open="false">
    Admins can create projects, manage users, configure integrations, and access all system features.
  </Expandable>
  <Expandable title="Editor Permissions" default-open="false">
    Editors can create, edit, and publish documentation within their assigned projects.
  </Expandable>
  <Expandable title="Viewer Permissions" default-open="false">
    Viewers can read documentation but cannot make changes or access administrative functions.
  </Expandable>
</ExpandableGroup>

## Security Best Practices

<Callout kind="alert">Always use HTTPS for API requests and store tokens securely.</Callout>

<Steps>
  <Step title="Use HTTPS" icon="shield">
    Ensure all API calls use HTTPS to encrypt data in transit.
  </Step>
  <Step title="Secure Tokens" icon="lock">
    Store API tokens in environment variables or secure vaults.
  </Step>
  <Step title="Rotate Keys" icon="refresh-cw">
    Regularly rotate API tokens and revoke unused ones.
  </Step>
  <Step title="Monitor Access" icon="eye">
    Monitor authentication logs for suspicious activity.
  </Step>
</Steps>

## Troubleshooting Authentication Issues

<Expandable title="Common Issues" default-open="false">
  - **Invalid Token**: Verify your API token is correct and not expired.
  - **Insufficient Permissions**: Check your user role and assigned permissions.
  - **Rate Limiting**: Monitor API usage to avoid hitting rate limits.
</Expandable>

<Callout kind="tip">Enable two-factor authentication for enhanced account security.</Callout>

## Integration Examples

<CodeGroup tabs="Node.js,Python">
  ```javascript
const karmix = require('karmix-alchemist-sdk');

const client = new karmix.Client({
  token: process.env.KARMIX_TOKEN,
  baseUrl: 'https://api.karmixalchemist.com/v1'
});

// Authenticate and fetch projects
client.authenticate().then(() => {
  return client.projects.list();
}).then(projects => {
  console.log('Projects:', projects);
});
  ```
  ```python
from karmix_alchemist import Client

client = Client(
    token=os.environ['KARMIX_TOKEN'],
    base_url='https://api.karmixalchemist.com/v1'
)

client.authenticate()
projects = client.projects.list()
print('Projects:', projects)
  ```
</CodeGroup>
